# 修复 app.json 缺失错误

## ❌ 错误信息

```
app.json:在项目根目录未找到 app.json
模拟器启动失败
```

## 🔍 问题原因

这是 Taro 项目，需要先编译才能生成小程序文件。`dist` 目录是空的，所以微信开发者工具找不到 `app.json`。

---

## ✅ 解决方案

### 步骤1：编译项目

在项目根目录运行：

```bash
pnpm dev:weapp
```

这会：
- 编译 Taro 项目
- 生成小程序文件到 `dist` 目录
- 包含 `app.json`、`app.js` 等必需文件
- 开启监听模式（文件变化时自动重新编译）

### 步骤2：确认编译成功

编译完成后，检查 `dist` 目录应该包含：
- `app.json` ✅
- `app.js` ✅
- `pages/` 目录 ✅
- 其他小程序文件 ✅

### 步骤3：在微信开发者工具中打开项目

1. **打开微信开发者工具**
2. **选择项目目录**：选择 `dist` 目录（不是项目根目录）
   - 项目路径应该是：`d:\tcm-bti-assessment\dist`
3. **AppID**：填写您的小程序 AppID（或选择"测试号"）
4. **点击"编译"**

---

## 📋 完整操作步骤

### 1. 停止当前编译（如果有）

如果之前有编译进程在运行，先按 `Ctrl+C` 停止。

### 2. 清理旧的编译文件（可选）

```bash
# 删除 dist 目录
rmdir /s /q dist
```

### 3. 重新编译

```bash
pnpm dev:weapp
```

等待编译完成，应该看到类似输出：
```
✔ 编译成功
```

### 4. 在微信开发者工具中打开

- **项目路径**：`d:\tcm-bti-assessment\dist`
- **AppID**：您的小程序 AppID
- **点击"编译"**

---

## ⚠️ 常见问题

### Q1: 编译失败

**检查：**
- Node.js 版本是否正确（建议 16+）
- 依赖是否安装完整：`pnpm install`
- 查看编译错误信息

### Q2: 仍然找不到 app.json

**检查：**
- 确认微信开发者工具指向的是 `dist` 目录
- 确认编译已成功完成
- 检查 `dist` 目录中是否有 `app.json` 文件

### Q3: 编译很慢

**优化：**
- 首次编译会较慢，后续会更快
- 可以关闭一些不必要的功能加速编译

## 📝 注意事项

1. **开发模式**：`pnpm dev:weapp` 会开启监听模式，文件变化时自动重新编译
2. **生产模式**：使用 `pnpm build:weapp` 进行生产环境编译
3. **项目路径**：微信开发者工具必须指向 `dist` 目录，不是项目根目录
