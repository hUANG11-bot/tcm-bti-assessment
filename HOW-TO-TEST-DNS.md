# 如何测试域名解析

## 🚀 方法1：使用 nslookup（推荐）

### 步骤1：打开命令行

1. **按 `Win + R` 键**
2. **输入 `cmd` 并按回车**
3. **或者按 `Win + X`，选择"Windows PowerShell"或"命令提示符"**

---

### 步骤2：执行 nslookup 命令

在命令行中输入：
```cmd
nslookup er1.store
```

然后按回车。

---

### 步骤3：查看结果

**如果解析成功**，您会看到类似这样的输出：
```
服务器:  UnKnown
Address:  192.168.1.1

非权威应答:
名称:    er1.store
Address:  123.456.789.012
```

**说明**：
- `名称: er1.store` - 域名
- `Address: 123.456.789.012` - 解析到的 IP 地址
- ✅ **如果看到 IP 地址，说明 DNS 解析正常**

---

**如果解析失败**，您会看到类似这样的输出：
```
服务器:  UnKnown
Address:  192.168.1.1

*** UnKnown 找不到 er1.store: Non-existent domain
```

**说明**：
- ❌ **如果看到"找不到"或"Non-existent domain"，说明 DNS 解析失败**
- 需要检查域名服务商的 DNS 配置

---

## 🔍 方法2：使用 ping 命令

### 步骤1：打开命令行

同上，打开 CMD 或 PowerShell。

---

### 步骤2：执行 ping 命令

在命令行中输入：
```cmd
ping er1.store
```

然后按回车。

---

### 步骤3：查看结果

**如果解析成功**，您会看到类似这样的输出：
```
正在 Ping er1.store [123.456.789.012] 具有 32 字节的数据:
来自 123.456.789.012 的回复: 字节=32 时间=50ms TTL=64
来自 123.456.789.012 的回复: 字节=32 时间=51ms TTL=64
```

**说明**：
- `[123.456.789.012]` - 解析到的 IP 地址
- ✅ **如果看到 IP 地址，说明 DNS 解析正常**

---

**如果解析失败**，您会看到类似这样的输出：
```
Ping 请求找不到主机 er1.store。请检查该名称，然后重试。
```

**说明**：
- ❌ **如果看到"找不到主机"，说明 DNS 解析失败**

---

## 🌐 方法3：在浏览器中测试

### 步骤1：打开浏览器

打开 Chrome、Edge 或其他浏览器。

---

### 步骤2：访问域名

在地址栏输入：
```
https://er1.store
```

然后按回车。

---

### 步骤3：查看结果

**如果能够访问**：
- 浏览器显示页面内容或服务器响应
- ✅ **说明域名解析正常，服务器也可以访问**

**如果无法访问**：
- 浏览器显示"无法访问此网站"或"DNS_PROBE_FINISHED_NXDOMAIN"
- ❌ **说明域名解析失败或服务器不可访问**

## ⚠️ 常见问题

### Q1: nslookup 显示"找不到"

**可能原因**：
1. DNS 配置还没有生效（需要等待几分钟到几小时）
2. DNS 配置错误（CNAME 或 A 记录配置不正确）
3. 域名服务商的 DNS 服务器有问题

**解决方法**：
1. 等待一段时间后重试
2. 检查域名服务商的 DNS 配置
3. 确认 CNAME 或 A 记录是否正确

---

### Q2: ping 可以解析，但浏览器无法访问

**可能原因**：
1. DNS 解析正常，但服务器没有运行
2. 防火墙阻止了访问
3. HTTPS 证书配置有问题

**解决方法**：
1. 检查服务器是否正常运行
2. 检查防火墙设置
3. 检查 HTTPS 证书配置

---

### Q3: 在线工具显示解析成功，但本地无法解析

**可能原因**：
1. 本地 DNS 缓存问题
2. 本地网络 DNS 服务器配置问题

**解决方法**：
1. 清除 DNS 缓存：
   ```cmd
   ipconfig /flushdns
   ```
2. 更换 DNS 服务器（如使用 8.8.8.8 或 114.114.114.114）
